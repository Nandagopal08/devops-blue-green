HIPAA Technical Safeguards — 45 CFR § 164.312
Access Control
Unique user IDs enforced. Automatic logoff after 15 minutes of inactivity. AES-256 encryption/decryption for all PHI at rest. mTLS for all service-to-service communication.
Audit Controls
All PHI access events logged to immutable S3 with object-lock. Splunk SIEM with pre-built HIPAA correlation rules. Real-time anomaly alerts for unusual access patterns. 7-year log retention.
Transmission Security
TLS 1.3 minimum — older versions rejected. AES-256-GCM at rest. FIPS 140-2 validated HSM for key management. WireGuard VPN required for all administrative access.
Zero-Trust Healthcare Architecture — Layered View
Zero-Trust Core Principles
Never Trust, Always Verify
No implicit trust based on network location. Every request authenticated via SMART-on-FHIR OAuth2 + mTLS. Short-lived tokens with continuous re-validation.
Least Privilege Access
RBAC enforced at Kubernetes + application layer. PHI access scoped to minimum necessary per HIPAA's Minimum Necessary Standard. Just-in-time privilege escalation via Vault.
Assume Breach Mindset
Micro-segmentation via Istio service mesh. Blast radius minimized by network policies. Automated isolation on anomaly detection. Continuous audit trail for forensics.
